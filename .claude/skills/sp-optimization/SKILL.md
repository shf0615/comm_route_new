# 代码优化

## 概述

在测试绿灯保护下优化代码。优化不能破坏任何行为。

**启动声明：** "我正在使用 sp-optimization 技能来优化代码。"

## 角色分离

### 主agent职责（你）
- 用户配置收集
- **循环控制**：评估子agent返回的结果，决定继续还是结束
- **决策**：恶性问题是否还存在？需要回滚吗？需要与用户确认吗？
- 每次迭代派遣**新的**子agent

### 子agent职责（每次新建）
- 接收明确的优化指令（范围 + 上轮遗留问题）
- 执行单轮优化
- 每项优化后运行测试
- **返回结构化结果**（不做循环决策）

## 前置条件

- 行为规格文件（`.feature`）— 作为行为守护基线
- 技术方案（`-design.md`）— 作为架构约束锚点
- 已通过全部测试的代码

## 流程

```
主agent: 用户配置 → 初始化状态
│
├─ 迭代 N:
│   ├─ 主agent: 构造子agent指令（含上轮遗留问题）
│   ├─ 派遣新子agent → 执行单轮优化 + 测试
│   ├─ 子agent返回: {已修复, 测试结果, 剩余恶性问题}
│   └─ 主agent决策:
│       ├─ 测试失败？→ 指示下轮子agent回滚
│       ├─ 还有恶性问题？→ 继续迭代
│       ├─ 需要改变行为？→ 暂停，问用户
│       └─ 无恶性问题 + 绿灯 → 结束
│
└─ 输出最终报告
```

## Step 1: 用户配置

可多选：
1. **架构** — 模块划分、耦合度、重复代码
2. **Bug/安全** — 潜在bug、资源泄漏、安全漏洞
3. **性能** — 运行效率、内存、响应速度
4. **可维护性** — 可读性、命名、代码组织

## Step 2: 迭代执行

### 主agent构造子agent指令模板：

```
优化范围：[用户选择的范围]
上轮遗留恶性问题：[列表，首次为空]
需要回滚的项：[列表，若有]

你的任务：
1. 针对上述范围执行一轮优化
2. 每项优化后运行全部测试
3. 测试失败则回滚该项
4. 返回结果（格式见下方）

返回格式：
- 已执行优化：[列表]
- 已回滚优化：[列表]
- 测试结果：全部通过 / 有失败（详情）
- 剩余恶性问题：[列表，无则为空]
- 需要用户确认的行为变更：[列表，无则为空]
```

### 主agent决策逻辑：

每次子agent返回后，主agent判断：
1. **有需要用户确认的行为变更** → 暂停循环，询问用户
2. **还有恶性问题** → 构造新指令，派遣新子agent继续
3. **无恶性问题 + 测试全过** → 结束循环

**恶性问题定义：** 功能bug、架构腐化、安全漏洞、资源泄漏

## Step 3: 结束与报告

循环结束后输出：
- 总迭代次数
- 所有已完成优化项
- 所有已回滚优化项
- 最终测试状态

## 约束

- 优化不能破坏任何 Scenario 测试
- 优化不能偏离技术方案的架构意图
- 行为变更必须先更新行为规格和测试
- **主agent不执行优化代码，只做决策和派遣**
- **子agent不做循环决策，只执行单轮并返回结果**
