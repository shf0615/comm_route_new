# 全面测试

## 概述

基于行为规格（`.feature`）编写并执行测试，确保所有 Scenario 被覆盖。

**启动声明：** "我正在使用 sp-testing 技能来进行全面测试。"

## 角色分离

### 主agent职责（你）
- 用户配置收集
- **循环控制**：维护通过计数，决定继续还是结束
- **决策**：是代码bug还是测试bug？通过计数是否归零？
- 每次迭代派遣**新的**子agent

### 子agent职责（每次新建）
- 接收明确的测试指令（含上轮失败信息）
- 编写/修复测试，执行一轮
- **返回结构化结果**（不做循环决策）

## 前置条件

- 行为规格文件（`.feature`）
- 已实现的代码

## 流程

```
主agent: 用户配置 → 初始化状态（通过计数=0）
│
├─ 迭代 N:
│   ├─ 主agent: 构造子agent指令（含上轮失败详情）
│   ├─ 派遣新子agent → 编写/修复测试 + 执行
│   ├─ 子agent返回: {测试结果, 修复类型, 失败详情}
│   └─ 主agent决策:
│       ├─ 全部通过？
│       │   ├─ 计数+1
│       │   └─ 达到连续通过次数？→ 结束
│       ├─ 有失败 + 修复了代码bug？→ 计数归零，继续
│       ├─ 有失败 + 修复了测试bug？→ 计数不归零，继续
│       └─ 有失败 + 未能修复？→ 继续（传递失败详情）
│
└─ 输出测试报告
```

## Step 1: 用户配置

### 测试层级（可多选）：
1. **单元测试** — 函数/模块级别
2. **集成测试** — 模块间交互
3. **端到端测试** — 完整 Scenario 流程

### 额外覆盖（可多选）：
1. **边界情况** — 极值、空值、溢出
2. **异常路径** — 错误输入、网络失败、超时
3. **性能** — 压力测试、响应时间
4. **安全** — 注入、越权、数据泄漏

### 连续通过次数：默认 3 次。

## Step 2: 迭代执行

### 主agent构造子agent指令模板：

**首次迭代：**
```
行为规格：[.feature 文件路径]
测试层级：[用户选择]
额外覆盖：[用户选择]

你的任务：
1. 从行为规格 Scenario 派生测试（一个 Scenario = 一个测试函数）
2. 从 Scenario Outline Examples 派生参数化测试
3. 根据额外覆盖配置补充测试
4. 执行全部测试
5. 有失败则尝试修复并重新执行（单轮内）
6. 返回结果（格式见下方）

测试编写规范：
- 测试结构映射 Given-When-Then（setup-act-assert）
- 每个测试标注来源：[Feature/Scenario名] / [额外-边界] 等

返回格式：
- 测试结果：全部通过 / 有失败
- 失败详情：[列表，无则为空]
- 修复记录：[修复了什么，是代码bug还是测试bug]
- 测试用例清单：[函数名 + 来源标注]
```

**后续迭代：**
```
上轮失败详情：[主agent传入]
上轮修复记录：[主agent传入]

你的任务：
1. 针对上轮失败项修复并重新执行全部测试
2. 返回结果（同上格式）
```

### 主agent决策逻辑：

维护状态：`连续通过计数`（初始 0）

每次子agent返回后：
1. **全部通过** → 主agent提交代码（git commit）→ 计数+1 → 达到目标？结束：继续
2. **有失败，修复了代码bug** → 计数归零 → 继续
3. **有失败，修复了测试bug** → 计数不变 → 继续
4. **有失败，未修复** → 计数不变 → 继续（传递失败详情给下轮）

## Step 3: 测试报告

循环结束后输出：

```markdown
## 测试报告

### 概览
- 总用例数：N
- Scenario 派生：X
- 额外补充：Y
- 总迭代次数：Z
- 最终状态：连续 M 次通过

### Scenario 覆盖
| Feature | Scenario | 测试函数 | 状态 |
|---------|----------|----------|------|
| ... | ... | ... | ✅ |

### 修复记录
- 迭代1：编写测试，发现 X 个失败
- 迭代2：修复代码bug Y，计数归零
- 迭代3-5：连续 3 次通过，结束
```

## 约束

- 行为规格中每个 Scenario 必须有对应测试，不可跳过
- 测试结构必须映射 Given-When-Then（setup-act-assert）
- 修复代码bug后通过计数归零
- **主agent不编写测试代码，只做决策和派遣**
- **子agent不做循环决策，只执行单轮并返回结果**
