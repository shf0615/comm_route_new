# 实现计划：引入统一内存池

## 设计总结

将当前 TX/广播/RX 的独立缓冲区替换为统一内存池。池由 `pool_size` 个大小为 `mtu` 字节的 block 组成。所有数据缓冲需求从池分配/释放。

**关键语义变化：** `mtu` 从"最大payload大小"改为"帧总大小"（5字节头 + payload），即 payload_per_frame = mtu - 5。

## 内存池设计

```
┌──────────────────────────────────────────────┐
│  block[0]  │  block[1]  │ ... │ block[N-1]   │  ← 每个 mtu 字节
└──────────────────────────────────────────────┘
┌──────────────────────────────────────────────┐
│  bitmap: 1 bit per block (0=free, 1=used)    │
└──────────────────────────────────────────────┘
```

- `cr_pool_alloc_n(self, n)` — 找到 N 个**连续**空闲 block，标记已用，返回首地址（NULL=失败）
- `cr_pool_free_n(self, ptr, n)` — 释放从 ptr 开始的 N 个连续 block
- 初始化时预留 1 个 block 作为 scratch（帧构建缓冲区），终身不释放

## 配置变更 (`cr_config_t`)

| 删除 | 新增 | 保留 |
|------|------|------|
| `tx_buf_per_slot` | `pool_size` (block总数) | `tx_queue_depth` |
| `rx_buf_per_slot` | `tx_max_blocks` (每个TX任务最大block数) | `rx_assem_count` |
| — | `rx_max_blocks` (每个RX slot最大block数) | `bcast_queue_depth` |
| — | — | `mtu` (语义改为帧总大小) |

## 各子系统变更

### 1. TX (`cr_send`)
- 计算所需 block 数: `n = ceil(len / mtu)`，不超过 `tx_max_blocks`
- 调用 `cr_pool_alloc_n(n)` 获取连续内存
- memcpy 用户数据到分配的区域
- `cr_tx_task_t` 新增字段: `uint16_t first_block; uint8_t block_count;`
- `task->data` 指向分配区域首地址（与当前逻辑一致）
- `cr_send_frame` 中 payload_per_frame = `mtu - CR_FRAME_HEADER_SIZE`
- 任务完成/失败时调用 `cr_pool_free_n` 归还 block
- 池满返回新错误码 `CR_ERR_NO_MEM`

### 2. 广播 (`cr_broadcast`)
- 限制: 单次广播不超过 `mtu - CR_FRAME_HEADER_SIZE` 字节（单帧，不分片）
- 分配 1 个 block，copy 数据
- `cr_tx_task_t` 同样用 `first_block + block_count` 跟踪
- 发送完成后释放

### 3. RX 重组
- `cr_rx_slot_t` 新增: `uint16_t first_block; uint8_t block_count;`
- 首个分片到达 → `cr_pool_alloc_n(rx_max_blocks)` 分配连续区域
- 后续分片写入 `pool_base + first_block * mtu + received_len`（与当前逻辑一致）
- 单帧消息（无 FRAG 标志）→ 直接从输入 data 投递，不分配 pool block
- 交付/超时 → `cr_pool_free_n` 释放
- 池满时丢帧（当前行为：无空闲 slot 时也丢帧）

### 4. Scratch（帧构建）
- `cr_init` 时从池预留 1 block → `self->tx_scratch`
- 用于 `cr_send_frame` 和 `cr_handle_broadcast_frame` 的帧转发
- 不释放，终身占用

## `cr_calc_buffer_size` 新计算

```
size = sizeof(cr_internal_t)
     + sizeof(cr_tx_task_t) * tx_queue_depth        // TX 队列管理结构
     + sizeof(cr_tx_task_t) * bcast_queue_depth     // 广播队列管理结构
     + sizeof(cr_rx_slot_t) * rx_assem_count        // RX slot 管理结构
     + sizeof(cr_dedup_entry_t) * dedup_table_size  // 去重表
     + mtu * pool_size                              // 内存池 blocks
     + (pool_size + 7) / 8                          // 位图
     + CR_ALIGN - 1                                 // 对齐余量
```

## `cr_internal_t` 变更

删除:
- `tx_buffers`, `bcast_buffers`, `rx_buffers`, `tx_scratch`

新增:
```c
/* 内存池 */
uint8_t  *pool_base;       // block 数组首地址
uint8_t  *pool_bitmap;     // 位图
uint16_t  pool_size;       // block 总数
uint16_t  pool_free_count; // 剩余可用 block 数
```

`tx_scratch` 保留为指针，但指向池中预留的 block。

## `cr_tx_task_t` 变更

删除: 无（`data` 指针保留，指向池 block）

新增:
```c
uint16_t first_block;  // 首 block 索引
uint8_t  block_count;  // 占用 block 数
```

## `cr_rx_slot_t` 变更

新增:
```c
uint16_t first_block;  // 首 block 索引
uint8_t  block_count;  // 占用 block 数
```

## `cr_finish_active_task` 变更

任务完成时释放 pool blocks:
```c
cr_pool_free_n(self, task->first_block, task->block_count);
```

## mtu 语义影响

当前代码中 `mtu` 作为 payload 上限使用的位置需要改为 `mtu - CR_FRAME_HEADER_SIZE`:
- `cr_send_frame`: `payload_len > self->cfg.mtu` → `payload_len > (self->cfg.mtu - CR_FRAME_HEADER_SIZE)`
- `cr_broadcast`: `len > self->cfg.mtu` → `len > (self->cfg.mtu - CR_FRAME_HEADER_SIZE)`
- `is_multi` 判断: `total_len > self->cfg.mtu` → `total_len > (self->cfg.mtu - CR_FRAME_HEADER_SIZE)`

## 实现步骤

1. **修改 `comm_route.h`**
   - 更新 `cr_config_t`: 删除 `tx_buf_per_slot`/`rx_buf_per_slot`, 新增 `pool_size`/`tx_max_blocks`/`rx_max_blocks`
   - 新增错误码 `CR_ERR_NO_MEM`（或在.c中内部定义）

2. **修改 `comm_route.c` — 数据结构**
   - 更新 `cr_internal_t`: 删除独立 buffer 指针，新增 pool 字段
   - 更新 `cr_tx_task_t`: 新增 `first_block`, `block_count`
   - 更新 `cr_rx_slot_t`: 新增 `first_block`, `block_count`

3. **修改 `comm_route.c` — 池操作**
   - 实现 `cr_pool_alloc_n()`
   - 实现 `cr_pool_free_n()`

4. **修改 `comm_route.c` — 初始化**
   - 重写 `cr_calc_buffer_size()`
   - 重写 `cr_init()` 的内存布局分配
   - 初始化时预留 1 block 给 scratch

5. **修改 `comm_route.c` — TX/广播/RX 逻辑**
   - `cr_send`: 用 pool_alloc_n 替代固定偏移
   - `cr_broadcast`: 同上
   - `cr_finish_active_task`: 释放 blocks
   - `cr_poll` 广播处理: 发送后释放 block
   - `cr_handle_local_frame` 分片处理: 用 pool 替代 rx_buffers
   - RX 超时: 释放 blocks
   - `cr_send_frame`: 修正 payload 计算（mtu - header）
